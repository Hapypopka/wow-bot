# WowBot — WoW 3.3.5a Bot для WoWCircle

## Что это
PvE бот (ротации, follow, хилбот, мультибоксинг) для WoW WoTLK 3.3.5a, приватный сервер WoWCircle. Русский клиент, x86.

## Сборка
```bash
dotnet publish WowBot.Injector -c Release -r win-x86 --self-contained -o publish
```
Publish блокируется если exe запущен — закрыть бот перед сборкой.

## Архитектура v2

### Модули (BotEngine делегирует, НЕ содержит логику):
- **CombatExecutor** — ЕДИНЫЙ бой для solo И slave (позиционирование + ротация)
- **BuffManager** — баффы (seal/blessing/aura/shout/stance/totems/pet)
- **CombatHelper** — подсчёт врагов, SmartTaunt, GroundAoE, AoE Avoidance
- **RotationRegistry** — поиск ротации по классу/спеку (C# → Lua fallback)
- **LuaHelpers** — единый набор Lua хелперов для всех ротаций

### Правило: куда добавлять код
- Боевые фичи → **CombatExecutor**, НЕ BotEngine.Tick()
- Баффы → **BuffManager**
- Новый спек → отдельный файл {Class}Rotation.cs

### Принципы
- **Никаких костылей** — корень проблемы, не симптом. Лучше потратить время на правильное решение
- **Один путь** — solo и slave используют один и тот же код (CombatExecutor)
- **Проверяй что код выполняется** — перед фиксом убедись что этот файл вообще загружается
- **Spell ID** — через GetSpellInfo(id), НЕ русские названия
- **ё → е** — в клиенте WoWCircle нет буквы ё

### Баг-воркфлоу (ОБЯЗАТЕЛЬНО)
Когда юзер жалуется на баг:
1. Найти причину в логах (`wowbot_<Char>_mark.log` если есть, иначе спросить timestamp) и/или коде
2. **Объяснить простым языком** — без "race condition", "override", "grace period". Как не-программисту
3. Предложить варианты фикса короткой фразой
4. **ЖДАТЬ явного "да, фиксим" / "делай"** — только потом Edit/Build
НЕ начинать сразу править код и рефакторить смежное — это несколько раз сносило нужный юзеру функционал.

## Ключевые технические детали
- Оффсеты WoWCircle ОТЛИЧАЮТСЯ от стандарта — см. `.claude/rules/wowcircle.md`
- GetLocalizedText КРАШИТ WoW — НЕ использовать
- Lua буфер 32KB, Tick каждые 150мс
- EndSceneHook — НЕ трогать без крайней необходимости
- NavEngine (AmeisenNavigation) — патхфайндинг через TCP сервер, fallback на CTM
