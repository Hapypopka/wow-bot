У брата (другой ПК) при Attach:
- DevicePtr2Offset 0x397C = NULL (стандартный оффсет не работает)
- Wow.exe тот же размер (7,704,216 байт) — клиент одинаковый
- D3D9Helper нашёл EndScene через скан (offset 0x2674), но он оказался ложным — все ExecuteLua таймаутили

Реализован автоскан с верификацией:
- Сканируем оффсеты 0x2000–0x5000 с шагом 4
- Для каждого кандидата: устанавливаем хук → тест `local x=1` → ждём 500мс
- Если таймаут → снимаем хук, пробуем следующего
- Пока не работает у брата — все кандидаты ложные

Возможные причины:
- Discord/GeForce/Steam оверлей подменяет D3D9 device
- OpenGL режим (SET gxApi "d3d9" в Config.wtf)

**Why:** Брат хочет пользоваться ботом, нужно универсальное решение
**How to apply:** При работе над D3D9 — учитывать что у разных ПК разные оффсеты
