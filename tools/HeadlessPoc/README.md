# HeadlessPoc — эксперимент сетевого WoW-клиента

## Что это
Минимальный POC: подключение к **logon-серверу** WoW 3.3.5a, авторизация через SRP6, получение списка реалмов. **Без world-сервера, без чтения мира, без кастинга.** Чисто чтобы пощупать протокол.

## Что НЕ делает
- Не подключается к world-серверу (там другая криптография — RC4 с ключом из K)
- Не читает персонажей (это уже world packets)
- Не двигает, не кастует, не парсит мир
- Не интегрирован с основным ботом — это отдельная песочница

## Зачем нужен
Проверить:
1. Работает ли SRP6 на нашем сервере (локальный SPP)
2. Видит ли сервер нашу версию билда (12340)
3. Отдаёт ли реалм-лист

Если **ЭТО** взлетит — следующий шаг (world server) сильно сложнее, но как минимум станет ясно что протокол доступен.
Если **НЕ** взлетит — значит к headless-варианту даже подходить не стоит.

## Запуск
```
cd tools/HeadlessPoc
dotnet run -- ACCOUNTNAME password 127.0.0.1 3724
```

По умолчанию `127.0.0.1:3724` — локальный SPP. Для WoWCircle — нужен их logon-адрес (мы его не используем в рамках POC).

## Ожидаемый вывод (успех)
```
[POC] Connecting to 127.0.0.1:3724 as 'FAIZ'
[POC] -> CMD_AUTH_LOGON_CHALLENGE (XX bytes)
[POC] <- challenge OK. B=AB12... salt=CD34...
[POC] SRP6 computed. A=...
[POC] -> CMD_AUTH_LOGON_PROOF (74 bytes)
[POC] <- proof OK. M2=EF56...
[POC] -> CMD_REALM_LIST
[POC] <- 1 realm(s):
  #505 'TEST' @ 127.0.0.1:8085  type=1 locked=0 ...
```

## Известные грабли
- `WOW_FAIL_INCORRECT_PASSWORD` — обычно проблема в SRP6 (порядок байт, регистр аккаунта/пароля). Аккаунт и пароль идут в `UPPER`.
- `WOW_FAIL_VERSION_INVALID` — сервер не принимает билд 12340. Менять `ClientBuild` в `Program.cs`.
- `EndOfStreamException` сразу после challenge — обычно сервер отбрасывает по таймауту или невалидной структуре пакета.

## Что дальше (если взлетит)
1. Подключение к world-серверу из адреса в realm-list
2. `SMSG_AUTH_CHALLENGE` → клиентское `CMSG_AUTH_SESSION` (RC4 шифрование с ключом K из SRP6)
3. `SMSG_CHAR_ENUM` — список персонажей
4. ...и **только потом** — мир, юниты, AoE, кастинг (~3-4 месяца работы)
