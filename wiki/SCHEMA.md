# WowBot Wiki — Schema

Это LLM-поддерживаемая вики проекта WowBot. Claude пишет и обновляет все страницы.
Человек управляет направлением, Claude делает бухгалтерию.

## Структура

```
wiki/
  index.md              — каталог всех страниц (обновлять при каждом изменении)
  log.md                — хронологический лог операций (append-only)
  overview.md           — обзор проекта, текущее состояние
  architecture/         — как устроены модули (CombatExecutor, Hivemind, BuffManager...)
  classes/              — страница на каждый класс WoW (ротации, баги, статус, спеллы)
  concepts/             — концепции (AoE, spell ID, Lua engine, позиционирование...)
  decisions/            — решения и их обоснования (WHY, не WHAT)
  sources/              — саммари внешних источников (NPCBots, TrinityCore, гайды)
```

## Правила для Claude

### Когда обновлять вики
- **После фикса бага** — обновить страницу класса/концепции, добавить в лог
- **После новой фичи** — создать/обновить страницы, добавить в index и лог
- **После архитектурного решения** — создать страницу в decisions/
- **После исследования** — создать саммари в sources/
- **НЕ обновлять** при мелких правках (опечатки, переименования)

### Формат страниц
- Markdown с Obsidian-совместимыми `[[wiki-links]]`
- Frontmatter YAML: `title`, `updated`, `tags`
- Ссылки между страницами через `[[имя-файла]]` без расширения
- Русский язык

### index.md
- Категории: Архитектура, Классы, Концепции, Решения, Источники
- Каждая запись: `- [[имя-файла]] — краткое описание`
- Обновлять при добавлении/удалении страниц

### log.md
- Формат: `## [YYYY-MM-DD] тип | Описание`
- Типы: `ingest`, `fix`, `feature`, `decision`, `research`, `lint`
- Append-only, новые записи сверху
- Кратко: что изменено, какие страницы затронуты

### Стиль
- Факты, не мнения. Конкретные spell ID, номера строк, имена файлов
- Если что-то сломано — писать прямо: "НЕ РАБОТАЕТ: причина"
- Если решение спорное — документировать альтернативы в decisions/
- Перекрёстные ссылки: если упоминаешь AoE на странице класса — `[[aoe-system]]`
