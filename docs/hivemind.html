Hivemind System

Мультибоксинг WowBot через аддон-сообщения WoW 3.3.5a

1. Обзор

Hivemind — система мультибоксинга, позволяющая одному игроку (мастеру) управлять несколькими ботами (слейвами) через внутриигровые аддон-сообщения. Не требует внешних программ или модификации клиента.

Принцип работы

WoW 3.3.5a предоставляет API SendAddonMessage(prefix, msg, channel), позволяющий аддонам обмениваться произвольными текстовыми сообщениями по каналу "PARTY" (группа/рейд). Hivemind использует канал с префиксом WBHIVE.

МАСТЕР (1 окно WoW) СЛЕЙВЫ (N окон WoW) | | |-- SendAddonMessage("WBHIVE", --|-- Lua listener ловит | "Attack:Lovelion#5", --| CHAT_MSG_ADDON | "PARTY") --| | |-- парсит CMD:arg#seq | |-- ExecuteSlaveCommand() | |-- SendAddonMessage("WBHIVE", | <-- ACK:5~Virtuella | "ACK:5~Virtuella", "PARTY") | |

Роли

РольEnumОписание
MASTERRole.MasterОтправляет команды, видит список слейвов, ловит ACK, Ctrl+ПКМ навигация

Ключевые классы

WB_SLAVE — при включении слейва устанавливается глобальная Lua-переменная WB_SLAVE=1. Она блокирует авто-героизм у слейвов (только по команде мастера через CastHeroism).

2. Протокол связи

Формат сообщений

Все сообщения передаются через SendAddonMessage с префиксом WBHIVE по каналу PARTY.

Формат основного сообщения

CMD:arg#seq
ЧастьОписаниеПример
CMDИмя команды (enum Command)Attack, Follow, Stop
argАргумент (имя мастера, координаты, и т.д.)Lovelion, 123.5;456.7;100.0
#seqSequence number для ACK (надёжная доставка)#5

Адресация (per-slave)

CMD:arg~targetList#seq

Если команда адресована конкретным слейвам, к аргументу добавляется ~имя1,имя2. Слейв проверяет своё имя в списке и игнорирует сообщение если его нет.

Специальные форматы

ТипФорматПример
RegisterRegister:имя;классRegister:Virtuella;PALADIN
ACKACK:seq~имяСлейваACK:5~Virtuella
SetBuffSetBuff:тип=ключSetBuff:blessing=BoM
GotoGoto:X;Y;ZGoto:123.5;456.7;100.0

SendCommand (мастер)

Метод SendCommand(Command cmd, string arg) — основной способ отправки команд.

SendCommand(Attack, "Lovelion") | +-- 1. Определить получателей | Есть Selected слейвы? | ДА -> targets = "Virtuella,Zabudi" | НЕТ -> Есть IgnoreGlobal слейвы? | ДА -> targets = все кроме IgnoreGlobal | НЕТ -> targets = "" (все получат) | +-- 2. Сформировать сообщение | _cmdSeq++ | fullArg = targets ? "Lovelion~Virtuella,Zabudi" : "Lovelion" | msg = "Attack:Lovelion~Virtuella,Zabudi#5" | +-- 3. Отправить через Lua | SendAddonMessage('WBHIVE','Attack:Lovelion~Virt...#5','PARTY') | +-- 4. Начать ACK tracking (если нужен) | _pendingSeq = 5 | _pendingAcks = { "Virtuella", "Zabudi" } | slave.AckStatus = "pending" | +-- 5. Обновить ActiveCommand для UI slave.ActiveCommand = Attack

Lua Listener (слейв)

При включении Hivemind устанавливается Lua-фрейм, слушающий событие CHAT_MSG_ADDON:

-- Устанавливается один раз (GetSlaveListenerScript)
WB_HIVE_FRAME = CreateFrame('Frame')
WB_HIVE_FRAME:RegisterEvent('CHAT_MSG_ADDON')
WB_HIVE_FRAME:SetScript('OnEvent', function(self, event, prefix, msg, channel, sender)
    if prefix ~= 'WBHIVE' then return end
    if sender == UnitName('player') then return end  -- свои сообщения игнорируем
    local cmd, arg = strsplit(':', msg, 2)
    if cmd == 'ACK' then
        WB_HIVE_ACK = arg          -- "5~Virtuella"
        WB_HIVE_ACK_TIME = GetTime()
    elseif cmd == 'Register' then
        WB_HIVE_REG = arg          -- "Virtuella;PALADIN"
        WB_HIVE_REG_SENDER = sender
        WB_HIVE_REG_TIME = GetTime()
    else
        WB_HIVE_CMD = cmd          -- "Attack"
        WB_HIVE_ARG = arg          -- "Lovelion#5"
        WB_HIVE_SENDER = sender
        WB_HIVE_TIME = GetTime()
    end
end)

C# читает переменные каждые 3 тика (450мс) через GetSlaveReadScript():

WB_R = WB_HIVE_CMD..'|'..WB_HIVE_ARG..'|'..WB_HIVE_SENDER..'|'..WB_HIVE_TIME

Результат парсится в ParseSlaveResponse() → кортеж (Command?, arg, sender, time).

Защита от чужих мастеров: Если у слейва уже есть MasterName, он игнорирует команды от других отправителей (проверка sender != MasterName).

3. ACK система

Обеспечивает гарантированную доставку команд. WoW аддон-сообщения могут теряться при нагрузке на сервер.

Поток ACK

МАСТЕР СЛЕЙВ | | |-- Send "Attack:Lovelion#5" --| | _pendingSeq=5 |-- ExecuteSlaveCommand() | _pendingAcks={"Virt","Zab"} | _slaveLastExecSeq = 5 | _retryCount=0 |-- SendAck(5) | AckStatus="pending" | "ACK:5~Virtuella" | | | <-- "ACK:5~Virtuella" | | ReceiveAck(5, "Virtuella") | | _pendingAcks={"Zab"} | | Virtuella.AckStatus="ok" | | | |--- 500мс без ACK от "Zab" --- | | TickAck() → retry #1 | | Re-send "Attack:Lovelion#5" --| | |-- seq==_slaveLastExecSeq | | Уже выполнено! Re-send ACK | <-- "ACK:5~Zab" | | ReceiveAck(5, "Zab") | | _pendingAcks={} DONE! | | |

Параметры

ПараметрЗначениеОписание
MaxRetries3Максимум повторных отправок
RetryIntervalMs500Интервал между retry (мс)
_slaveLastExecSeqintПоследний выполненный seq на слейве

Дедупликация на слейве

Слейв хранит _slaveLastExecSeq. При получении команды:

  • seq < _slaveLastExecSeq — устаревшая, игнорировать
  • seq == _slaveLastExecSeq — уже выполнена, повторно отправить ACK
  • seq > _slaveLastExecSeq — новая, выполнить + отправить ACK

AckStatus (per-slave)

ЗначениеОписание
""Нет активного ожидания
"pending"Команда отправлена, ждём ACK
"ok"ACK получен
"timeout"3 retry исчерпаны, ACK не получен

Команды без ACK

Служебные команды не требуют подтверждения:

SetBuff, Wipe, Interact, GossipSelect, GossipAccept, AutoToggleFollow, AutoToggleAttack, Ping

5. Режимы слейвов (SlaveMode)

РежимОписаниеFollowРотацияБафы
IdleОжидание (после Stop или при старте)НетЕсли есть таргет*Нет
FollowingБежит за мастеромДа (CTM)Да (instants на бегу, full стоя)Да
AttackingБьёт таргет мастераНетДа + approachДа
AutoFollow + авто-ассистВне бояВ боюДа
GoingToPointБежит к Ctrl+ПКМ точкеК точкеДа (instants на бегу)Да

* В Idle бьёт только если ForceIdle == false и _rotationEnabled == true.

Idle

Начальное состояние. Слейв стоит на месте. Если ForceIdle=true (после Stop) — полностью молчит. Если ForceIdle=false и ротация включена — бьёт текущий таргет (если жив, в бою, не игрок).

case SlaveMode.Idle:
    if (ForceIdle || !_rotationEnabled) break;
    // Бьём таргет если есть (face + rotation)
    var idleTarget = _objectManager.GetTarget();
    if (target alive && inCombat && !isPlayer)
        FaceInstant(player, target);
        ExecuteLua(fullScript);

Following

Слейв бежит за мастером через SlaveController.Tick() (CTM или NavMesh). Пока бежит — кастует только instants. Когда добежал (стоит) — полная ротация + поворот к таргету.

Following tick (150мс) | +-- SlaveCtrl.Tick() // движение к мастеру | +-- player стоит? | НЕТ (бежит): | Есть враг-таргет? -> Lua: instants only | ДА (добежал): | Есть враг-таргет? | -> TryGroundAoE() | -> FaceInstant() | -> Lua: полная ротация

Attacking

Слейв атакует таргет. Подбегает через Lua MoveForwardStart()/MoveForwardStop(). Поддерживает MoveBehind (заход за спину) и RangedPos (позиционирование ранж).

Attacking tick (SlaveAttackTick) | +-- AoE Avoidance (выбежать из лужи?) | +-- Есть таргет? Жив? В бою? | НЕТ -> остановить approach, return | +-- Игрок кастует? -> return | +-- MoveBehind / RangedPos (если включено) | +-- FaceInstant (поворот к таргету) | +-- TryGroundAoE (Гроза / Семя порчи) | +-- Approach через Lua: | if not CheckInteractDistance('target', distId) | then MoveForwardStart() | else MoveForwardStop() | +-- ExecuteLua(approachLua + enemyCountLua + spellFlags + fullScript)

Параметр distId: 3 для мили (ближний бой), 1 для ранж (кастеры).

Auto

Гибридный режим: вне боя — follow за мастером, в бою — атака таргета мастера (авто-ассист каждые ~1 сек). Подробности в разделе Авто-режим подробно.

GoingToPoint

Слейв бежит к заданной точке (Ctrl+ПКМ мастера). Логика 1:1 как Following, но вместо мастера — координаты. Используется SlaveCtrl.CmdGotoPoint() с поддержкой NavMesh.

6. Авто-режим подробно

Тоглы авто-режима

ТоглКомандаЭффект
AutoPauseFollowAutoToggleFollowПауза follow: слейв стоит на месте, но бьёт
AutoPauseAttackAutoToggleAttackПауза атаки: слейв бежит за мастером, но не бьёт + ClearTarget()

SlaveAutoTick

Вызывается из BotEngine.Tick() только когда AutoPauseAttack == false. Каждые ~1 сек (7 тиков) делает AssistUnit(masterName) для получения таргета мастера.

public void SlaveAutoTick()
{
    if (Mode != SlaveMode.Auto || string.IsNullOrEmpty(MasterName)) return;
    _autoAssistTick++;
    if (_autoAssistTick < 7) return;  // ~1 сек (7 * 150мс)
    _autoAssistTick = 0;

    // AssistUnit берёт CommandSourceName (мастер), не FollowTarget
    string assistName = _botEngine?.SlaveCtrl.CommandSourceName ?? MasterName;
    _hook.ExecuteLua($"ClearTarget() AssistUnit('{assistName}')");
}

Логика Auto тика в BotEngine

Auto tick (BotEngine) | +-- BossEngine активен? -> тактика управляет | +-- SlaveAutoTick() // ассист каждые ~1с | +-- Есть таргет? (жив, в бою, не игрок, AutoPauseAttack=false) | ДА (В БОЮ): | HivemindFollowing? -> false, SlaveCtrl.CmdStop() | -> SlaveAttackTick() // подбег + ротация | | НЕТ (ВНЕ БОЯ): | AutoPauseFollow? -> пропуск | !HivemindFollowing? -> CmdFollow(masterName) | -> SlaveCtrl.Tick() // follow за мастером

Важно: При переключении из боя в follow (и обратно) система корректно останавливает/запускает CTM движение через SlaveCtrl.CmdStop() / SlaveCtrl.CmdFollow().

7. Команды мастера

Follow

ПараметрЗначение
EnumCommand.Follow
АргументИмя мастера (например, "Lovelion")
ACKДа
Метод мастераCmdFollow()

Что делает слейв:

  1. Устанавливает MasterName = arg
  2. Переключает Mode = Following
  3. SlaveCtrl.CmdFollow(masterName) — начинает CTM к мастеру
  4. HivemindFollowing = true
  5. Авто-включает ротацию и бафы через OnAutoToggle
  6. НЕ трогает таргет — продолжает бить текущую цель

Attack

ПараметрЗначение
EnumCommand.Attack
АргументИмя мастера
ACKДа
Метод мастераCmdAttack()

Что делает слейв:

  1. Устанавливает MasterName
  2. Mode = Attacking
  3. HivemindFollowing = false
  4. SlaveCtrl.CmdStop() — остановить follow
  5. Lua: AssistUnit('мастер') StartAttack() — берёт таргет мастера
  6. Авто-включает ротацию и бафы

Stop

ПараметрЗначение
EnumCommand.Stop
Аргументнет
ACKДа
Метод мастераCmdStop()

Что делает слейв:

  1. Mode = Idle, ForceIdle = true
  2. BotEngine.ForceStop() — выключает ротацию, follow, бафы
  3. SlaveCtrl.CmdStop() — CTM на свою позицию (мгновенная остановка)
  4. Lua: ClearTarget()
  5. Авто-выключает ротацию и бафы через OnAutoToggle("rotation", false)

Auto

ПараметрЗначение
EnumCommand.Auto
АргументИмя мастера
ACKДа
Метод мастераCmdAuto()

Что делает слейв:

  1. Mode = Auto
  2. AutoPauseFollow = false, AutoPauseAttack = false
  3. SlaveCtrl.CmdFollow(masterName) — follow мастера
  4. HivemindFollowing = true
  5. Авто-включает ротацию и бафы
  6. Далее BotEngine автоматически переключает follow/attack по наличию таргета

Scatter

ПараметрЗначение
EnumCommand.Scatter
АргументДистанция в метрах (по умолчанию 12)
ACKДа
Методы мастераCmdScatter(meters), CmdSmartScatter()

Что делает слейв:

  1. Вычисляет уникальный угол: hash(имяСлейва) → baseAngle + facing * 0.3
  2. Рассчитывает точку: X + dist*cos(angle), Y + dist*sin(angle)
  3. CTM.MoveTo(targetX, targetY, Z)
  4. НЕ меняет Mode — продолжает текущий режим (атака/авто)

Каждый слейв бежит в свою сторону благодаря уникальному хэшу имени.

Мастер читает gossip-опции через Lua: GetGossipOptions() (до 20 пунктов). На время чтения GossipReading=true — hivemind polling приостанавливается (чтобы не перезаписать WB_R).

Сервисные команды

SetBuff

Формат: SetBuff:тип=ключ (например blessing=BoM, aura=AuRet). НЕ отменяет текущую команду слейва. Вызывает OnBuffChanged в UI.

Wipe

Переключает WipeMode: хилеры перестают хилить (используется для контролируемых вайпов).

RefreshGuid

Аргумент — имя игрока. Слейв сбрасывает кэшированный GUID и ищет заново через SlaveCtrl.SetFollowTarget(name). Меняет ТОЛЬКО follow target, НЕ command source.

CastHeroism

Адресуется первому шаману среди слейвов. Слейв кастует spell ID 32182 (Героизм) через GetSpellInfo(32182).

AutoToggleFollow / AutoToggleAttack

Работают только в Auto режиме. Переключают AutoPauseFollow / AutoPauseAttack. При паузе follow — SlaveCtrl.CmdStop(). При паузе атаки — ClearTarget().

Ping

Слейв отвечает Pong:имя через addon message.

8. Per-slave команды

Механизм адресации

Есть два способа отправить команду конкретному слейву:

1. Через Selected (основной)

В UI мастер выбирает слейвов (checkbox). SendCommand() автоматически добавляет ~имя1,имя2 к аргументу. Слейв парсит arg.Split('~', 2) и проверяет targetList.Contains(myName).

2. Через SendCommandToSlave (прямая адресация)

public void SendCommandToSlave(string slaveName, Command cmd)
{
    string fullArg = $"{arg}~{slaveName}";
    string msg = $"{cmd}:{fullArg}";
    SendAddonMessage('WBHIVE', msg, 'PARTY');
}

Используется для: per-slave Follow/Attack/Stop, RefreshGuid, SetBuff.

3. IgnoreGlobal

Если у слейва IgnoreGlobal=true, общие (неадресные) команды автоматически адресуются только слейвам без этого флага. Это позволяет, например, оставить одного слейва на месте, пока остальные бегут.

9. SlaveController

Управляет движением слейва. Вызывается из BotEngine.Tick() каждые 150мс.

Состояния

StateОписание
IdleНе двигается
FollowingБежит за мастером (CTM или NavMesh)
GoingToPointБежит к заданным координатам
StoppedОстановлен (через 3 сек -> Idle)
Attacking(определён но не используется в Tick)

Follow логика (TickFollowing)

TickFollowing (150мс) | +-- FindMaster() -> GUID -> WowUnit | +-- dist = player.DistanceTo(master) | +-- dist <= FollowDistance? | ДА -> CTM.Stop(), NavEngine.Stop() | _isFollowMoving = false "arrived" | | НЕТ: | Вычислить goalX/goalY: | Точка на FollowDistance от мастера, в сторону игрока | goal = master.pos + normalize(player - master) * followDist | | NavMesh подключён? | ДА -> NavEngine.NavigateTo(goal) | Мастер сдвинулся < 5м? -> NavEngine.Tick() (не пересчитывать) | Мастер сдвинулся >= 5м -> Новый путь | NavMesh fail? -> Fallback CTM | | НЕТ -> CTM.MoveTo(goal) | Мастер сдвинулся < 3м и CTM активен? -> пропустить

CmdStop (мгновенная остановка)

public void CmdStop()
{
    // CTM на свою позицию — мгновенная остановка (CTM перебивает CTM)
    var player = _objectManager.LocalPlayer;
    _ctm.MoveTo(player.X, player.Y, player.Z, 0.5f);
    // Lua: остановить все движения
    _hook.ExecuteLua("MoveForwardStop() MoveBackwardStop() StrafeLeftStop() StrafeRightStop()");
    CurrentState = State.Stopped;
    _stopTimer = 20;  // 3 сек → Idle
}

Определение ролей происходит в Lua-скрипте ротаций (AllRotations.cs), а не в Hivemind. Однако это напрямую влияет на поведение слейвов-танков.

local function TryTaunt(tauntId)
    if not WB_S or WB_S.AutoTaunt==false then return false end
    if not IR(tauntId) then return false end       -- не готов
    if not UnitAffectingCombat('target') then return false end

    local tv = UnitName('targettarget')
    if not tv then return false end

    if isMT then
        if tv ~= myName then Cast(tauntId) return true end
    elseif isOT and WB_MT then
        if tv ~= myName and tv ~= WB_MT then Cast(tauntId) return true end
    else
        if tv ~= myName then Cast(tauntId) return true end
    end
    return false
end

11. Ctrl+ПКМ навигация

Мастер может отправить слейвов в точку зажав Ctrl и кликнув ПКМ по земле. Реализовано через быстрый поток (5мс), следящий за CTM памятью.

Как работает

CtmWatch Thread (5мс цикл) | +-- Ctrl зажат? | НЕТ -> обновить baseline (lastCtmX/Y/Z) | | ДА -> CTM action == MoveTo? | НЕТ -> пропуск | ДА -> Координаты сдвинулись > 1м? | НЕТ -> пропуск | ДА: | 1. CTM мастера на свою позицию | (отменить движение мастера) | 2. _gotoFired = true | _gotoX/Y/Z = координаты клика | --- следующий тик BotEngine (150мс) --- | +-- MasterTick(): if _gotoFired: _gotoFired = false CmdGoto(_gotoX, _gotoY, _gotoZ)

Антиход мастера: При Ctrl+ПКМ мастер НЕ бежит к точке — его CTM перебивается на свою же позицию (_ctm.MoveTo(cachedPlayerX, cachedPlayerY, cachedPlayerZ, 0.5f)). Бегут только слейвы.

StartCtmWatch / StopCtmWatch

StartCtmWatch() запускается автоматически при CurrentRole = Master. Останавливается при смене роли. Поток фоновый (IsBackground = true).

12. Выбор слейвов для навигации

Мастер может выбрать конкретных слейвов для Ctrl+ПКМ навигации через dropdown/контекстное меню.

КоллекцияТипОписание
NavSelectedSlavesHashSet<string>Слейвы выбранные для следующего Goto
NavPinnedSlavesHashSet<string>Закреплённые — не снимаются после Goto

Поведение

  • Если NavSelectedSlaves не пусто — Goto идёт только выбранным
  • После Goto незакреплённые автоматически снимаются: NavSelectedSlaves.RemoveWhere(n => !NavPinnedSlaves.Contains(n))
  • Закреплённые остаются и получат следующий Goto тоже
  • Если NavSelectedSlaves пусто — используется SendCommand(Goto) для всех
