1. Обзор продукта

Бизнес-требования

ТребованиеОписаниеСтатус
БТ-1: Групповое управление Мастер управляет 1+ слейвами в пати/рейде WoW. Один клик — все слейвы исполняют. Готово
БТ-2: Следование Слейвы физически бегут за мастером, держат дистанцию 8м. Возобновляют после боя. Готово
БТ-3: Ассист и атака По команде слейвы берут таргет мастера и выполняют полную боевую ротацию. Готово
БТ-4: Авто режим Гибрид: следование вне боя, авто-ассист при входе в бой. Бесшовные переходы. Готово
БТ-5: Экстренная остановка Мгновенный стоп: остановка движения, сброс таргета, переход в idle. Должно работать всегда. Проблема
БТ-6: Надёжная доставка Команды должны доставляться и подтверждаться. Повтор при неудаче. Статус по каждому слейву. Готово
БТ-7: Горячие клавиши Назначение F1-F12 / 0-9 / A-Z как глобальных хоткеев для Атаки, Ко мне, Стоп. Работают без фокуса. Готово

Технические требования

ТребованиеРеализация
ТТ-1: Неактивируемое окно WS_EX_NOACTIVATE | WS_EX_TOOLWINDOW — клик по панели НЕ крадёт фокус у WoW
ТТ-2: Дочернее окно WoW SetParent(masterPanelHwnd, wowHwnd) — панель привязана к окну WoW
ТТ-3: Канал связи SendAddonMessage('WBHIVE', ..., 'PARTY') — встроенный аддон-канал WoW
ТТ-4: Частота тиков Таймер BotEngine: 150мс. Обработка команд каждый тик.
ТТ-5: Единый ProcessCommand И UI, и Hivemind команды проходят через BotEngine.ProcessCommand()
ТТ-6: ACK протокол Sequence numbers, повтор 500мс, макс. 3 попытки, статус по каждому слейву

2. Спецификация команд

Четыре основные команды формируют главный контрольный цикл. У каждой есть UI-кнопка, опциональный хоткей и путь через Hivemind broadcast.

🏃

Ко мне (Follow)

BotCommandType.Follow — "follow"

Бизнес-смысл

Все слейвы физически бегут к мастеру и следуют за ним, держа дистанцию ~8м. Ротация и баффы включаются автоматически — слейвы готовы к бою на ходу.

Точки вызова

ИсточникПуть в коде
UI кнопкаBtnFollow_Click() → SendCommand("follow")
ХоткейWM_HOTKEY (HK_FOLLOW=2) → SendCommand("follow")
Hivemind (слейв)CHAT_MSG_ADDON → парсинг Command.Follow

Изменение состояния (слейв)

ПеременнаяДоПосле
Hivemind.ModeлюбойSlaveMode.Following
Hivemind.MasterName-имя персонажа мастера
HivemindFollowingfalsetrue
_rotationEnabledfalsetrue (авто)
BuffsEnabledfalsetrue (авто)

Логика движения

  • Целевая точка: позиция мастера + 8м смещение в направлении слейва
  • Порог обновления: пересчёт CTM если мастер сдвинулся >3м
  • Навигация: NavEngine (навмеш) если подключён, иначе прямой CTM
  • Условие остановки: расстояние до мастера ≤ 8м
  • Реализация: SlaveController.TickFollowing() → CTM.MoveTo()

Соло режим

Без Hivemind: просто ставит _followEnabled = true и запускает FollowViaCTM() с кружением на 8м вокруг таргета.

⚔

Бейте таргет (Attack)

BotCommandType.Attack — "attack"

Бизнес-смысл

Слейвы берут таргет мастера через ассист и бьют полной ротацией. Следование останавливается — чистый боевой режим. Для фокус-файра по приоритетной цели.

Точки вызова

ИсточникПуть в коде
UI кнопкаBtnAttack_Click() → SendCommand("attack")
ХоткейWM_HOTKEY (HK_ATTACK=1) → SendCommand("attack")
Hivemind (слейв)CHAT_MSG_ADDON → парсинг Command.Attack

Изменение состояния (слейв)

ПеременнаяДоПосле
Hivemind.ModeлюбойSlaveMode.Attacking
HivemindFollowingtruefalse (follow ВЫКЛ)
_rotationEnabled-true
BuffsEnabled-true

Последовательность исполнения

  1. SlaveCtrl.CmdStop() — остановка текущего движения
  2. ExecuteLua("AssistUnit('мастер') StartAttack()") — взять таргет мастера, начать автоатаку
  3. Каждый тик (150мс): SlaveAttackTick() → CombatExecutor.ExecuteCombatTick()
  4. CombatExecutor: позиционирование (MoveBehind/RangedPos), Ground AoE, SmartTaunt, Lua ротация
  5. Скрипт подхода: if not CheckInteractDistance('target', 3) then MoveForwardStart()... (мили)

Ключевые Lua операции

-- Сразу при получении команды
AssistUnit('Lovelion')     -- Взять таргет мастера
StartAttack()               -- Начать автоатаку

-- Каждый тик (скрипт ротации)
enemyCountLua + spellFlagsLua + rotationScript  -- ~500мс Lua
🔄

Авто

BotCommandType.Auto — "auto"

Бизнес-смысл

Режим "настроил и забыл". Слейвы следуют за мастером вне боя, автоматически ассистят при входе в бой, бесшовно возвращаются к следованию после боя. Два под-тогла позволяют независимо паузить follow или attack.

Точки вызова

ИсточникПуть в коде
UI кнопкаBtnAuto_Click() → SendCommand("auto")
Hivemind (слейв)CHAT_MSG_ADDON → парсинг Command.Auto

Важно: У Авто НЕТ хоткея. Только Атака, Ко мне и Стоп имеют хоткеи.

Изменение состояния (слейв)

ПеременнаяДоПосле
Hivemind.ModeлюбойSlaveMode.Auto
AutoPauseFollow-false (сброс)
AutoPauseAttack-false (сброс)
HivemindFollowing-true (начать follow)
_rotationEnabled-true
BuffsEnabled-true

Логика тика Авто (SlaveAutoModeTick)

Каждые 150мс: 1. SlaveAutoTick() раз в ~1 сек: AssistUnit('мастер') // взять таргет мастера 2. Оценка: hasAutoTarget = таргет жив И в бою И !AutoPauseAttack 3a. hasAutoTarget = true → В БОЮ - Остановить follow если активен: SlaveCtrl.CmdStop() - Бой: SlaveAttackTick() → CombatExecutor.ExecuteCombatTick() 3b. hasAutoTarget = false → ВНЕ БОЯ - Возобновить follow: SlaveCtrl.CmdFollow(мастер) - Движение: SlaveCtrl.Tick() → CTM к мастеру

Под-тоглы

ТоглКомандаЭффект
Пауза Follow AutoToggleFollow Переключает AutoPauseFollow. Если на паузе: не двигается вне боя. Если снята: CTM к мастеру.
Пауза Attack AutoToggleAttack Переключает AutoPauseAttack. Если на паузе: ClearTarget(), не бьёт. Если снята: авто-ассист.
⏹

Стоп

BotCommandType.Stop — "stop"

Бизнес-смысл

Экстренный тормоз. Вся активность прекращается: движение стоп, таргет сброшен, слейв в Idle. Используется для предотвращения вайпов, случайных пулов, перегруппировки.

Точки вызова

ИсточникПуть в коде
UI кнопкаBtnStop_Click() → SendCommand("stop")
ХоткейWM_HOTKEY (HK_STOP=3) → SendCommand("stop")
Hivemind (слейв)CHAT_MSG_ADDON → парсинг Command.Stop

Изменение состояния (слейв)

ПеременнаяДоПосле
Hivemind.ModeлюбойSlaveMode.Idle
Hivemind.ForceIdlefalsetrue (блок боя)
HivemindFollowingtruefalse

Последовательность исполнения (слейв)

  1. Hivemind.Mode = Idle, ForceIdle = true
  2. SlaveCtrl.CmdStop():
    • CTM в свою позицию (мгновенный стоп через CTM-override)
    • Lua: MoveForwardStop() MoveBackwardStop() StrafeLeftStop() StrafeRightStop()
    • Состояние → Stopped → Idle (через 3 сек / 20 тиков)
  3. ExecuteLua("ClearTarget()") — сброс таргета

Соло режим

Ставит _followEnabled = false и вызывает StopFollowMovement() (CTM.Stop). Пытается StopTimer() если больше ничего не активно.

3. Поток данных: Кнопка → Исполнение на слейве

Полный пайплайн команды

[UI кнопка / Хоткей] ↓ MasterPanel.SendCommand("attack") ↓ OnCommand?.Invoke("attack") MainWindow.OnMasterCommand("attack") ↓ ЕСЛИ Hivemind активен: ↓ Hivemind.SendCommand(Command.Attack, "имяМастера") ↓ seq++, msg = "Attack:имяМастера#1" ↓ Трекинг ACK: _pendingAcks = {"слейв1", "слейв2"} ↓ Lua: SendAddonMessage('WBHIVE', 'Attack:имяМастера#1', 'PARTY') ↓ --- аддон-канал WoW --- ↓ [Слейв] событие CHAT_MSG_ADDON ↓ WB_HIVE_CMD = "Attack", WB_HIVE_ARG = "имяМастера#1" ↓ [Тик слейва] ParseSlaveResponse() ↓ ExecuteSlaveCommand(Attack, "имяМастера#1") ↓ Парсинг seq=1, проверка > _slaveLastExecSeq ↓ SendAck(1): SendAddonMessage('WBHIVE', 'ACK:1~имяСлейва', 'PARTY') ↓ BotEngine.ProcessCommand(BotCommand { Type=Attack }) ↓ switch(cmd.Type) → case Attack: ↓ Hivemind.Mode = Attacking ↓ SlaveCtrl.CmdStop() ↓ ExecuteLua("AssistUnit('мастер') StartAttack()") ↓ _rotationEnabled = true, BuffsEnabled = true ↓ EnsureRunning() ↓ [Каждые 150мс] BotEngine.Tick() ↓ SlaveAttackTick() → CombatExecutor.ExecuteCombatTick() ↓ Позиционирование + Lua ротация → _hook.ExecuteLua(rotationScript, 500)

Архитектура очередей

Две очереди команд дренятся каждый тик в ProcessPendingCommands():

ОчередьИсточникНаполняется
UserCommands UI кнопки, хоткеи UserCommandSource.EnqueueCommand()
HivemindCommands Аддон-сообщения от мастера HivemindCommandSource.EnqueueCommand() через ExecuteSlaveCommand()

Обе очереди → один ProcessCommand(). Без приоритетов. Порядок FIFO.

4. ACK система — Гарантированная доставка

Обзор протокола

ПараметрЗначение
Интервал повтора500мс
Макс. попыток3 (общий таймаут: 1.5 сек)
Формат сообщения{Команда}:{аргумент}#{seq} — напр. Follow:Lovelion#42
Формат ACKACK:{seq}~{имяСлейва} — напр. ACK:42~Virtuella
ТранспортSendAddonMessage('WBHIVE', msg, 'PARTY')
Защита от дублейСлейв хранит _slaveLastExecSeq. Дубль → повторный ACK, без повторного исполнения.

Таймлайн: Успешная доставка

T+0мс — Мастер: seq++, отправка "Attack:master#1" через аддон-канал. Трекинг _pendingAcks = {"Virtuella"}
T+~50мс — Слейв: событие CHAT_MSG_ADDON. WB_HIVE_CMD = "Attack"
T+~200мс — Тик слейва: ParseSlaveResponse() читает команду. ExecuteSlaveCommand(Attack, "master#1")
T+~250мс — Слейв: seq=1 > lastExec=0. Исполнение команды. SendAck(1): "ACK:1~Virtuella"
T+~350мс — Мастер: ReceiveAck(1, "Virtuella"). Удаление из pending. AckStatus = "ok" (зелёный в UI)

Таймлайн: Повтор и таймаут

T+0мс — Мастер отправляет. ACK не получен.
T+500мс — TickAck(): повтор #1. Повторная отправка.
T+1000мс — TickAck(): повтор #2. Повторная отправка.
T+1500мс — TickAck(): повтор #3 (макс). Сдаёмся. AckStatus = "timeout" (красный в UI)

Команды без ACK (fire-and-forget)

Эти команды отправляются один раз без повтора. UI может показать "выполнено", но слейв мог не получить:

  • SetBuff, Wipe, Interact, GossipSelect, GossipAccept
  • AutoToggleFollow, AutoToggleAttack, Ping

6. Система горячих клавиш

КомандаID хоткеяДоступные клавишиПо умолчанию
Бейте таргетHK_ATTACK = 1F1-F12, 0-9, A-ZНет
Ко мнеHK_FOLLOW = 2F1-F12, 0-9, A-ZНет
СтопHK_STOP = 3F1-F12, 0-9, A-ZНет

Техническая реализация

  • Регистрация: Win32 RegisterHotKey() — глобальные, работают даже когда фокус на WoW
  • Перехват: WndProc ловит WM_HOTKEY (0x0312)
  • Назначение: Клик по кнопке в панели настроек → GetAsyncKeyState() опрос каждые 50мс
  • Отмена: Нажать Escape для отмены назначения
  • Сохранение: Через GetHotkeySettings() в settings.json, восстановление через LoadHotkeySettings()
  • Сброс: Кнопка "Сбросить" снимает регистрацию и чистит словарь

Процесс назначения

Юзер кликает "Нажми..." (напр., Атака) ↓ BtnSetHotkey_Click() ↓ _waitingForHotkey = "attack" ↓ Старт DispatcherTimer (опрос 50мс) ↓ CaptureTimer_Tick(): проверка GetAsyncKeyState(F1..F12, 0..9, A..Z) ↓ Клавиша нажата → AssignHotkey(key) ↓ _hotkeys["attack"] = Key.F5 ↓ RegisterAll() → RegisterHotKey(handle, 1, 0, VK_F5) ↓ UpdateHotkeyLabels() → UI показывает "F5" под кнопкой Атаки

8. Найденные проблемы и пробелы

Аудит актуального кода выявил следующие проблемы. Отсортированы по серьёзности.

Критично #1: Стоп НЕ выключает ротацию/баффы

Файл: BotEngine.cs:264-281

Проблема: Стоп ставит Hivemind.Mode = Idle и ForceIdle = true, сбрасывает таргет — но НЕ ставит _rotationEnabled = false и BuffsEnabled = false. Они были включены предыдущей командой Follow/Attack/Auto.

Последствия: После Стопа слейв физически стоит, но таймер движка крутится. Если таргет каким-то образом появится (Tab, проксимити), слейв может кастануть спеллы. Нарушение БТ-5 (экстренная остановка).

Фикс: Добавить _rotationEnabled = false; BuffsEnabled = false; в ветку Hivemind Stop (после строки 271).

Критично #2: Атака без таргета — тихий фейл

Файл: BotEngine.cs:294-303

Проблема: AssistUnit('мастер') StartAttack() выполняется безусловно. Если у мастера нет таргета, AssistUnit ничего не делает — слейв входит в режим Attacking с null target.

Последствия: Слейв в режиме Attacking, боевой цикл крутится каждый тик, жрёт Lua бюджет, но ничего не делает. Нет обратной связи. Надо вручную жать Стоп.

Фикс: После AssistUnit проверить наличие таргета. Если нет — не переключать в Attacking, или авто-откат в Follow.

Критично #3: Нет приоритета команд в очереди

Файл: BotEngine.cs:200-211

Проблема: Обе очереди (UserCommands и HivemindCommands) работают по FIFO. Команда Стоп ждёт за поставленными в очередь SetBuff/Interact.

Последствия: В панической ситуации с кучей команд в очереди, Стоп может задержаться на 1+ тик (150мс+ на каждую команду перед ним). Для "экстренного тормоза" это слишком медленно.

Фикс: Приоритет: Stop/Emergency прыгают в начало очереди. Или: отдельный флаг _emergencyStop, проверяемый до обработки очереди.

Серьёзно #4: Счётчик авто-ассиста дрифтит между режимами

Файл: Hivemind.cs, SlaveAutoTick()

Проблема: _autoAssistTick инкрементится глобально (все тики), а не только в режиме Авто. При переключении Follow → Auto счётчик может быть на произвольном значении.

Последствия: Первый авто-ассист после смены режима может сработать мгновенно или задержаться до 1 сек. Непредсказуемое время реакции.

Фикс: Сброс _autoAssistTick = 0 при входе в Авто (в ProcessCommand, case Auto).

Серьёзно #5: Дисконнект мастера — слейв молча стоит

Файл: SlaveController.cs, FindMaster()

Проблема: Если мастер уходит из зоны видимости или дисконнектится, FindMaster() возвращает null. Движение молча останавливается. Ни ошибки, ни фоллбека.

Последствия: Слейв выглядит зависшим. Юзер не понимает почему. Надо заново жать Ко мне.

Фикс: Таймаут. Если мастер не найден N тиков — лог предупреждение, опционально событие "мастер потерян" в UI.

Серьёзно #6: ACK подтверждает получение, а не исполнение

Файл: Hivemind.cs, ExecuteSlaveCommand()

Проблема: ACK отправляется сразу при парсинге команды, ДО запуска ProcessCommand(). Если ProcessCommand кинет исключение или слейв крашнется — мастер видит "ok", но команда не выполнена.

Последствия: Ложное подтверждение. Юзер думает слейвы бьют, а они упали после ACK.

Фикс: Отправлять ACK после завершения ProcessCommand, или добавить периодический heartbeat.

Минор #7: Таймаут ACK слишком короткий (1.5 сек)

Файл: Hivemind.cs, MaxRetries = 3, RetryIntervalMs = 500

Проблема: 3 попытки × 500мс = 1.5 сек. На лаговых серверах или медленных ПК аддон-сообщения могут идти 200-400мс. Легитимная доставка может протаймаутить.

Последствия: Ложный "timeout" в UI. Юзер повторяет команды, вызывая дубли (защита seq есть, но путает).

Минор #8: Вайп влияет только на хилов

Файл: Hivemind.WipeMode

Проблема: Название "Вайп" подразумевает остановку всех. На деле только хилы перестают хилить. ДПС продолжает бить.

Последствия: Путаница. При намеренном вайпе ДПС продолжает фармить и может случайно убить босса.

Минор #9: Навигация схлопывается при обновлении слейвов

Файл: MasterPanel.xaml.cs, UpdateSlaves()

Проблема: UpdateSlaves() пересоздаёт весь SlavePanel. Если дропдаун навигации был раскрыт, он схлопывается — состояние _navExpanded не применяется к пересозданным элементам.

Последствия: Юзеру надо каждый раз заново раскрывать навигацию (обновление каждые несколько секунд).

Минор #10: У Авто нет хоткея

Файл: MasterPanel.xaml.cs — только HK_ATTACK, HK_FOLLOW, HK_STOP

Проблема: Авто — самый используемый режим, но хоткея нет. Юзеру приходится кликать по панели.

Последствия: Неудобство. Авто должен быть 4-м хоткеем.
