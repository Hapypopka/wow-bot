1. Оверлей -- обзор OverlayWindow.xaml.cs -- прозрачный оверлей поверх WoW

Что это

Прозрачное WPF-окно, которое отображается поверх клиента WoW 3.3.5a. Главная особенность: окно имеет стиль WS_EX_NOACTIVATE (константа 0x08000000), что означает, что клики по оверлею не забирают фокус у WoW. Игрок может нажимать кнопки в оверлее, не теряя управление персонажем.

Визуальная структура

Оверлей состоит из трёх слоёв:

ЭлементНазначениеПоведение
MainButtonОсновная кнопка (круглая)Клик = открыть/закрыть меню. Drag = перетащить окно. Позиция сохраняется в настройках.
MenuPanelГлавное меню (вертикальный список)Пункты: Rotation, AoE, Buffs, Follow, Target, Hivemind, Reload, Lua
SubPanel / SubContentПодменю (контент секции)Перестраивается динамически при выборе пункта меню

Пункты меню

ПунктМетодОписание
RotationBuildRotationSubmenu()Тоглы спеллов + бурсты + слайдеры маны
AoEBuildAoeSubmenu()AoE тоглы по классу + слайдер мин. врагов
BuffsBuildBuffsSubmenu()Баффы, радио-выборы (печати, ауры, тотемы...)
FollowBuildFollowSubmenu()Цель следования + слайдер дистанции
TargetBuildTargetSubmenu()Автоповорот, автовыбор, забегание за спину, AoE уклонение
HivemindBuildHivemindSubmenu()Мультибоксинг: мастер/слейв, команды, список слейвов
ReloadOnReloadScriptsПеречитать Lua скрипты с диска (hot-reload)
LuaLuaToggle_ClickLua консоль: всплывающее окно для выполнения произвольных команд

Как работают тоглы

Каждый тогл -- это ToggleButton 34x34 пикселя с иконкой спелла. Состояние (IsChecked) собирается в таблицу WB_S через метод GetSpellFlagsLua() и передаётся в Lua-скрипт ротации перед каждым выполнением.

// Пример вывода GetSpellFlagsLua():
WB_S={VT=true,DP=true,SWP=true,MB=true,MF=true,SF=true,Disp=true}

2. Раздел Ротация

Метод BuildRotationSubmenu() строит подменю ротации на основе текущего спека игрока. Спеллы берутся из словаря SpecSpells, бурсты -- из SpecBursts.

Алгоритм построения

  1. Определить specKey по текущему спеку (напр. "Shadow Priest")
  2. Найти массив спеллов в SpecSpells[specKey]
  3. Для каждого спелла создать иконку 34x34 в WrapPanel
  4. Состояние загружается из сохранённых настроек или используется дефолт
  5. Добавить секцию бурстов из SpecBursts[specKey] (по дефолту OFF)
  6. Добавить специфичные для спека слайдеры (мана Dispersion, мана SF и т.д.)
  7. Для танковых спеков добавить секцию "Танкование" (AutoTaunt, DefCD, AoEThreat)
  8. Для паладинов добавить выбор правосудия (радио JoW / JoL)

4. Раздел AoE

Метод BuildAoeSubmenu(). Глобальный слайдер WB_AEMIN (мин. врагов для AoE, диапазон 2-10, дефолт 3) доступен для всех классов. Количество врагов подсчитывается в C# (CountNearbyCombatEnemies) и передаётся в Lua как WB_NCE.

Генерация Lua скрипта (BuildBuffScript)

Метод BuildBuffScript() в BotEngine генерирует однострочный Lua-скрипт (многострочный триггерит taint WeakAuras). Порядок проверок:

  1. Аура паладина -- HasB('player', 'Аура ...')
  2. Печать паладина -- с AoE свапом (SoV <-> SoC при 2+ врагах)
  3. Благословение паладина -- на себя, затем на пати/рейд. Великое с реагентом "Знак королей"
  4. Крик воина -- HasB('player', 'Боевой крик')
  5. Стойка воина -- GetShapeshiftForm()
  6. Власть ДК -- GetShapeshiftForm()
  7. Форма друида -- GetShapeshiftForm()
  8. Тотемы шамана -- SetMultiCastSpell(slot, spellId) для Зова Духов
  9. Аспект охотника -- авто-переключение дракондор/гадюка по мане (вне боя: <100%, в бою: <30%/>80%)
  10. Камень чар (варлок) -- проверка чары на оружии, создание + применение
  11. Пет варлока -- авто-призыв вне боя, dismiss + resummon при смене
  12. Пет охотника -- авто-призыв/воскрешение пета
  13. ДК: гуль + Костяной щит
  14. Оружие шамана (МН + ОН) -- GetWeaponEnchantInfo(), отдельная обработка OH через PickupInventoryItem(17)
  15. Self-баффы -- простая проверка HasB('player', name)
  16. Рейд-баффы -- проверка себя + всех в пати/рейде, с реагентами и fallback на одиночные

7. Ротации -- как работает

Цикл BotEngine (Tick)

BotEngine содержит таймер с интервалом ~150мс. Каждый тик:

AntiAFK (каждые ~30с)
→
ObjectManager.Update()
→
AoE Avoidance (DynObjects)
→
Buff check (каждый N-й тик)
→
Follow movement (CTM)
→
Smart Taunt (ObjectManager)
→
Combat Positioning
→
Ground AoE
→
FaceTarget
→
ExecuteLua(rotation)

Сборка скрипта ротации

Перед выполнением ротации собирается строка Lua-кода:

// Формула:
string script = enemyCountLua + SpellFlagsLua + GetRotationScript(player);

// enemyCountLua:
WB_NCE=5 WB_AEMIN=3    -- количество врагов + порог AoE

// SpellFlagsLua (из OverlayWindow.GetSpellFlagsLua()):
WB_S={VT=true,DP=true,...,CoA=true,SoV=true,...}

// GetRotationScript -- либо fullScript, либо AoE multidot скрипт

Обёртка скриптов

-- DPS скрипт оборачивается в:
local function WB_Run()
    -- Helpers (SN, Cast, IR, HB, HD, NR, BS, MP, THP, PHP, CP...)
    -- MainTank определение (WB_MT)
    -- TryTaunt, TryDefCD, TryRes...
    -- PreChecksDPS:
    if IsMounted() then return end
    if UnitIsDeadOrGhost('player') then return end
    -- Break-CC проверка
    if UnitCastingInfo / UnitChannelInfo then return end
    if not UnitAffectingCombat('target') then return end
    -- ... проверки таргета ...
    RunMacroText('/startattack') -- каждую секунду
    -- РОТАЦИЯ КЛАССА
end
local ok,err=pcall(WB_Run)
if not ok then WB_ERR=err print('WB_ERR: '..tostring(err)) end

10. MainTank определение

Определяется через GetRaidRosterInfo с кешем 5 секунд (глобальные переменные WB_MT, WB_MT_T).

-- Кеш: обновляем каждые 5с
if not WB_MT or not WB_MT_T or GetTime()-WB_MT_T>5 then
    WB_MT=nil WB_MT_T=GetTime()
    local nr=GetNumRaidMembers()
    if nr>0 then
        for i=1,nr do
            local n,_,_,_,_,_,_,_,_,role=GetRaidRosterInfo(i)
            if role=='MAINTANK' then WB_MT=n break end
        end
    end
end

11. Настройки

Файл настроек

Путь: {exe_dir}/settings_{CharName}.json (или settings.json если имя неизвестно). Формат: JSON, pretty-printed.

Что сохраняется

КатегорияКлючи
Позиция окнаpos_x, pos_y
Радио-выборыcurse, pet, seal, blessing, judgement, aura, shout, stance, presence, feralForm, totemEarth, totemFire, totemWater, totemAir
Основные тоглыaoe, buffs
Спелл-тоглыspell_{Key} (напр. spell_VT, spell_Meta)
Бафф-тоглыbuff_{SpellName} (напр. buff_Молитва стойкости)
Слайдерыslider_dispMana, slider_sfMana, slider_maxDots, slider_mindSear, slider_aoeMin, slider_dist, slider_maxRange, slider_defHP
Чекбоксыchk_autoFace, chk_autoTarget, chk_moveBehind, chk_aoeAvoid, chk_multiDot, chk_mindSear, chk_defAll
AoE тоглыaoe_SoC

Поток загрузки/сохранения

  1. SetPlayerClass() вызывается при определении класса/спека
  2. Загружает settings_{charName}.json через LoadSettings()
  3. Очищает все тоглы (_spellToggles.Clear(), _buffToggles.Clear())
  4. Устанавливает дефолты радио-выборов по классу/спеку из сохранённых или хардкод
  5. Pre-create тоглы из SpecSpells, SpecBursts, ClassBuffs с сохранёнными значениями
  6. При любом изменении (клик, перетаскивание) -- SaveSettings() сериализует всё в JSON
Если UI-элемент ещё не создан (подменю не открывалось), значение берётся из _saved словаря -- предыдущее сохранённое значение не теряется при перезаписи.
