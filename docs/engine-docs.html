BotEngine Game/BotEngine.cs

Главный координатор бота. Запускает таймер на 150мс, который вызывает метод Tick(). Отвечает за: обновление ObjectManager, follow-логику, выполнение ротаций, бафф-скрипты, AoE avoidance, smart taunt, AntiAFK и координацию Hivemind.

Tick() - Поток выполнения

Каждые 150мс вызывается метод Tick(object? state). Вот полный порядок:

→
Проверка: активен ли бот?
→

→

→
SetPlayerBase()
→
AoE Avoidance
→

Далее выполнение зависит от режима:

Приоритет выполнения

  1. AntiAFK - всегда, даже если бот выключен
  2. AoE Flee - если IsAoeFleeing == true, пропускаем ВСЁ на 2 секунды
  3. AoE Avoidance (TryAoEAvoidance) - DynObject детекция, flee из лужи
  4. Hivemind Slave - если роль Slave, полностью управляется SlaveController
  5. Hivemind Master - MasterTick(), ACK чтение, Register
  6. Баффы - BuildBuffScript / BuildClassBuffScript
  7. Solo Follow - FollowViaCTM (если только follow без rotation)
  8. Solo Rotation - FaceInstant + ExecuteLua (ротация)
  9. Follow + Rotation - CTM follow + instants на бегу, полная ротация на месте
private void Tick(object? state)
{
    if (_tickPaused) return;
    AntiAfkTick();  // ВСЕГДА, даже если бот неактивен

    if (!_followEnabled && !_rotationEnabled && !_buffsEnabled && !Hivemind.IsActive) return;
    if (!_hook.IsHooked) return;

    _objectManager.Update();
    var player = _objectManager.LocalPlayer;
    if (player == null) return;

    // AoE Avoidance: ПРИОРИТЕТ НАД ВСЕМ
    if (IsAoeFleeing) return;
    if (AoeAvoidEnabled && TryAoEAvoidance(player)) return;

    // ... Hivemind, Buffs, Follow, Rotation ...
}

Solo режим

Когда Hivemind НЕ активен, бот работает в соло-режиме. Три подрежима:

Только Follow

_followEnabled && !_rotationEnabled -- Бот бежит за целью через FollowViaCTM(). Считает точку на земле в _followDistance от цели в сторону игрока и вызывает CTM.MoveTo(). Не спамит CTM если цель стоит.

Только Rotation

!_followEnabled && _rotationEnabled -- Поворот к таргету (FaceInstant), затем Lua ротация. Включает: SmartTaunt, MoveBehind/RangedPos, Ground AoE, автовыбор таргета (ассист танка или TargetNearestEnemy).

Follow + Rotation

Комбинированный режим: если далеко -- бежим (CTM follow) + instants на ходу. Если на месте -- полная ротация + face target. При касте -- CTM.Stop().

Slave режим (Hivemind)

Когда Hivemind.CurrentRole == Role.Slave, BotEngine передает управление SlaveController. Режимы слейва:

РежимКодПоведение
FollowingSlaveMode.FollowingSlaveCtrl.Tick() рулит движением. На бегу -- instants. На месте -- полная ротация + face.
GoingToPointSlaveMode.GoingToPointИдентично Following, но к Ctrl+ПКМ точке.
AttackingSlaveMode.AttackingSlaveAttackTick(): AoE avoidance, face, approach через Lua, MoveBehind, ротация.
AutoSlaveMode.AutoВ бою: attack. Вне боя: follow за мастером. Авто-ассист таргета мастера.
IdleSlaveMode.IdleНичего не делать (ForceIdle) или бить текущий таргет если есть.

Хилер-слейв -- особый случай

Хилер ВСЕГДА хилит (каждые ~450мс), независимо от SlaveMode. Follow продолжается параллельно с хилом. Единственное исключение -- WipeMode.

Система баффов

Два типа проверки баффов:

  • BuildClassBuffScript() -- быстрая проверка каждые 0.45с (3 тика). Стойка воина, власть ДК, форма друида, аура палладина, печать палладина.
  • BuildBuffScript() -- полная проверка каждые 3с (20 тиков). Все self-баффы, рейд-баффы (с реагентами), благословения, шаманские оружия/тотемы.

Всё генерируется как однострочный Lua -- многострочный триггерит taint WeakAuras.

Рейд-баффы с реагентами

Система проверяет реагент через GetItemCount(). Если реагент есть -- кастует рейд-бафф. Если нет -- fallback на одиночный бафф (напр. "Молитва стойкости" → "Слово силы: Стойкость").

AntiAFK

Вызывается каждый тик, но действует каждые ~30 секунд (200 тиков).

Двойная защита

  1. Запись в память: WriteUInt32(0x00B499A4, Environment.TickCount) -- LastHardwareAction (WoWCircle: сдвинут на +8 от стандарта)
  2. Lua снятие: if UnitIsAFK('player') then SendChatMessage('','AFK') end -- если AFK уже стоит, сбрасываем

Невидимый для сервера: нет движения, нет анимации, нет побочных эффектов.

AoE / Ground AoE

CountNearbyEnemies

Два метода подсчета врагов:

  • CountNearbyEnemies(player, range) -- все живые юниты в радиусе
  • CountNearbyCombatEnemies(player, range) -- только в бою, таргетящие союзников
  • CountEnemiesNearTarget(target, range) -- враги рядом с таргетом (для AoE решений)

Результат передается в Lua как WB_NE, WB_NCE, WB_AEMIN.

TryGroundAoE

Ground-targeted AoE (Гроза, Залп). Алгоритм:

  1. Проверить: AoE включен, не кастуем, не ченнелим
  2. Подсчитать врагов в 10м от таргета (CountEnemiesNearTarget)
  3. Если врагов ≥ AoeMinEnemies -- каст спелла через Lua
  4. Thread.Sleep(100) -- дать WoW время перейти в режим прицеливания
  5. _hook.CastTerrainClick(target.X, target.Y, target.Z) -- клик на позицию таргета

EndScene Hook Game/EndSceneHook.cs

Ядро внедрения: перехватывает вызов IDirect3DDevice9::EndScene в WoW, перенаправляя его в codecave с нашим ассемблерным кодом. Codecave проверяет флаг и выполняет действия (Lua, TerrainClick, CTM) в главном потоке WoW.

D3D9Helper (fallback)

Если стандартная цепочка сломана (оверлеи Discord, GeForce и т.д.), используется fallback через фейковое D3D9 устройство.

Алгоритм

  1. Создаем скрытое окно через CreateWindowExA("STATIC", "WB_D3D", ...)
  2. Создаем D3D9: Direct3DCreate9(32)
  3. CreateDevice: adapter=0, HAL, SOFTWARE_VERTEXPROCESSING. Минимальные параметры (1x1 backbuffer)
  4. Читаем VTable[42] -- адрес EndScene в нашем процессе
  5. Вычисляем смещение: offset = EndScene_local - d3d9.dll_local
  6. Ищем d3d9.dll в WoW: через CreateToolhelp32Snapshot + Module32First/Next
  7. Применяем смещение: EndScene_remote = d3d9.dll_remote + offset

Signature matching (крайний fallback)

Если d3d9.dll не найден по имени -- перебираем ВСЕ модули процесса WoW. Для каждого читаем base+offset и сравниваем первые 4 байта с нашим EndScene. Это ловит ренейменные/кастомные d3d9.

Install / CalculateStolenBytes

Установка хука: подменяем первые байты EndScene на JMP в наш codecave.

CalculateStolenBytes

Минимум 5 байт нужно украсть (размер JMP rel32). Но нельзя резать инструкции пополам. Мини-дизассемблер разбирает x86 инструкции:

FixupStolenBytes

Относительные инструкции (call/jmp rel32) нужно пересчитать, т.к. codecave по другому адресу:

target = originalAddr + offset + 5 + oldRel
newRel = target - (newAddr + offset + 5)

// Патч:     E9 xx xx xx xx [NOP...]

ASM логика codecave

pushad                      ; сохраняем все регистры
pushfd                      ; сохраняем флаги

mov eax, [_flagAddr]         ; читаем флаг
cmp eax, 1                  ; flag == 1? (exec lua)
je  exec_lua
cmp eax, 3                  ; flag == 3? (exec + read)
je  exec_result
cmp eax, 4                  ; flag == 4? (read only)
je  read_only
cmp eax, 5                  ; flag == 5? (terrain click)
je  terrain_click
cmp eax, 6                  ; flag == 6? (CTM call)
je  ctm_call
jmp skip                    ; flag == 0 -- ничего не делать

exec_lua:
  push 0                    ; Lua_DoString(luaStr, "bot", 0)
  push _botStringAddr
  push _luaStringAddr
  call Lua_DoString          ; 0x00819210
  add esp, 12
  mov [_flagAddr], 2         ; flag = done
  jmp skip

exec_result:
  ; Lua_DoString + lua_getfield + lua_tolstring + lua_settop
  ... call Lua_DoString ...
  push "WB_R"
  push LUA_GLOBALSINDEX     ; -10002
  push [LuaState]           ; 0x00D3F78C
  call lua_getfield          ; 0x0084E590
  ...
  call lua_tolstring         ; 0x0084E0E0 -- возвращает char* в eax
  mov [_returnPtrAddr], eax
  call lua_settop(L, 0)     ; 0x0084DBF0 -- очистка стека
  mov [_flagAddr], 2

terrain_click:
  push _terrainClickAddr
  call HandleTerrainClick    ; 0x00527830
  add esp, 4
  mov [_flagAddr], 2

ctm_call:
  push [precision]           ; struct+24
  push posAddr               ; struct+12 (X,Y,Z)
  push guidAddr              ; struct+4
  push [clickType]           ; struct+0
  mov ecx, [playerBase]      ; struct+28 (this pointer)
  call 0x00727400            ; CGPlayer_C__ClickToMove
  mov [_flagAddr], 2

skip:
  popfd
  popad
  ; [stolen bytes -- оригинальные инструкции EndScene]
  jmp endScene + stolenByteCount  ; возврат

Протокол взаимодействия

C# записывает данные в буферы, затем ставит флаг. ASM в EndScene видит флаг, выполняет действие, ставит flag=2 (done). C# в цикле WaitForCompletion() ждёт flag=2, затем сбрасывает в 0.

ExecuteLua / ExecuteLuaWithResult

ExecuteLua (flag=1)

  1. Записать Lua-строку в _luaStringAddr
  2. Установить _flagAddr = 1
  3. WaitForCompletion: поллить flag каждые 1мс, ждать flag==2
  4. Сбросить flag=0, вернуть true

ExecuteLuaWithResult (двухшаговый)

  1. Шаг 1: ExecuteLua (flag=1) -- выполняет Lua, устанавливает WB_R
  2. Пауза 50мс -- ждём 1 EndScene кадр
  3. Шаг 2: flag=4 (read only) -- lua_getfield("WB_R") + lua_tolstring
  4. Читает указатель из _returnPtrAddr
  5. Читает строку по указателю: ReadString(resultPtr, 4096)

WaitForCompletion

while (elapsed < timeout)
{
    flag = ReadUInt32(_flagAddr);
    if (flag == 2) { WriteUInt32(_flagAddr, 0); return true; }
    if (flag == 0) return false;  // кто-то уже сбросил
    Thread.Sleep(1);
}
// timeout -- сбрасываем flag и возвращаем false

Верификация хука

После установки хука выполняется тестовый скрипт:

_memory.WriteString(_luaStringAddr, "local x=1");
_memory.WriteUInt32(_flagAddr, 1);
WaitForCompletion(1000);  // timeout 1 секунда

Если флаг не стал 2 за 1 секунду -- хук считается нерабочим, вызывается Uninstall() и выбрасывается исключение.

Ауры (GetAuraSpellIds)

Таблица аур хранится по двум путям:

  • Обычный путь: count @ base+0xDD0, таблица @ base+0xC50
  • Альтернативный (count == -1): count @ base+0xC54, таблица -- указатель @ base+0xC58

Каждая аура: размер 0x18 (24 байта), spell ID @ offset 0x08 внутри записи. Макс. 80 аур.

public bool FaceInstant(WowUnit player, WowUnit target)

    if (IsFacing(player, target)) return true;

AoE Avoidance В BotEngine.cs

Автоматическое выбегание из вражеских AoE зон (луж). Основан на DynObject детекции.

Алгоритм (TryAoEAvoidance)

  1. Проверка каждые ~450мс (3 тика)
  2. Получить список DynObjects из ObjectManager
  3. Получить ауры (дебаффы) игрока
  4. Для каждого DynObject: SpellId совпадает с дебафом на нас?
  5. Проверить SafeAoeDebuffs (Осквернение ДК -- только слоу, не бежим)
  6. Расстояние до центра DynObject < radius + 2yd?
  7. Если несколько луж -- вычислить центр масс
  8. Вектор побега: от центра масс наружу на maxRadius + 5yd
  9. CTM.MoveTo() к точке побега
  10. Grace period: _aoeFleeUntil = +2 секунды
escapeAngle = atan2(player.Y - meanY, player.X - meanX)
escapeDist  = maxRadius + 5
fleeX = meanX + escapeDist * cos(escapeAngle)
fleeY = meanY + escapeDist * sin(escapeAngle)

Приоритет: AoE Flee имеет наивысший приоритет в Tick(). Пока IsAoeFleeing == true, пропускаются: follow, rotation, hivemind, всё.

LuaReader Game/LuaReader.cs

Читает результаты Lua через макрос WoW. Записывает значение в макрос через EditMacro(), затем читает строку из памяти WoW.

Initialize (двухпроходный скан)

  1. Создание макроса: CreateMacro('WB', 1, 'init') если нет
  2. Проход 1 (маркер A): EditMacro(1, 'WB', 1, 'WBMA_12345_PADDING...')
  3. Скан памяти: ищем маркер A в 3 регионах (0x01-0x08, 0x08-0x20, 0x20-0x30 GB)
  4. Проход 2 (маркер B): записываем другой маркер
  5. Верификация: среди кандидатов из прохода 1 ищем кто содержит маркер B
  6. Найденный адрес = _macroAddr

Зачем 2 прохода?

Маркер A может быть найден в нескольких местах (Lua буфер, копия строки, кеш). Проход 2 перезаписывает только настоящий адрес макроса -- остальные кандидаты устаревают.

LuaConcatMarker

Маркер разбивается конкатенацией: 'WBM'..'A_12345' -- чтобы полная строка НЕ попала в Lua буфер (иначе будет ложный кандидат).

Execute

  1. Очистить макрос: EditMacro(1,'WB',1,'WB_CLR')
  2. Проверить адрес (прочитать 'WB_CLR' по _macroAddr)
  3. Если адрес протух -- быстрый пересканм
  4. ExecuteLua(luaCode) -- код должен заканчиваться EditMacro(1,'WB',1,WB_R)
  5. Thread.Sleep(300) -- дать WoW время
  6. ReadString(_macroAddr, 255)
  7. Если пусто -- одна попытка retry (+200мс)

Автопереинициализация

Если 5 подряд неудач (MAX_FAILS_BEFORE_REINIT), вызывается повторная Initialize().
