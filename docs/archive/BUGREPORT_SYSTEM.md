# Система баг-репортов и авто-обновлений WowBot

## Цель
Дать брату и другу (не разработчики) возможность:
1. Находить баги и отправлять отчёты
2. Claude автоматически фиксит
3. Они обновляют бота одной кнопкой
4. Если фикс плохой — откатывают

## Архитектура

### Поток данных
```
Пользователь (брат/друг)          Сервер (Faiz)              GitHub
        │                              │                        │
        ├─ "Сова не кастует МФ" ──────>│                        │
        │   (UI кнопка или Telegram)   │── создаёт Issue ──────>│
        │                              │                        │
        │                              │<── webhook/poll ───────│
        │                              │── Claude Code fix ────>│
        │                              │── commit + push ──────>│
        │                              │── build publish.zip ──>│
        │                              │── GitHub Release ─────>│
        │                              │── закрывает Issue ────>│
        │                              │                        │
        │<── "Доступно обновление!" ───│                        │
        ├─ нажимает "Обновить" ───────>│                        │
        │<── скачивает publish.zip ────│<── Release asset ──────│
        │                              │                        │
        ├─ нажимает "Откатить" ───────>│                        │
        │<── скачивает prev release ───│<── Previous Release ───│
```

## Вариант 1: Всё в UI бота (WPF)

### UI в оверлее (новая секция "Баги")
```
┌─────────────────────────────────┐
│  Нашел баг?                     │
│  ┌─────────────────────────┐    │
│  │ Опиши проблему...       │    │
│  │ Например: "На совухе    │    │
│  │ не кастуется Лунный     │    │
│  │ огонь после Eclipse"    │    │
│  └─────────────────────────┘    │
│  Спек: Balance Druid (авто)     │
│  [Отправить баг]                │
│                                 │
│  Версия: v1.2.3                 │
│  [Проверить обновления]         │
│  [Откатить на предыдущую]       │
│                                 │
│  Статус: Всё актуально ✓        │
└─────────────────────────────────┘
```

### Что прикрепляется к баг-репорту автоматически
- Текущий спек (Balance Druid, Holy Paladin и т.д.)
- Версия бота
- Последние 50 строк из wowbot.log
- Текущие тоглы (WB_S={...})
- Имя персонажа

### Технические детали
- **GitHub Issues API**: `POST /repos/{owner}/{repo}/issues`
- **GitHub PAT**: зашит в приложение (scope: repo) или в config
- **GitHub Releases API**: `GET /repos/{owner}/{repo}/releases/latest`
- **Авто-обновление**: скачать zip → распаковать → перезапустить

## Вариант 2: Telegram бот

### Плюсы
- Им привычнее (все знают телегу)
- Можно кидать скриншоты
- Не нужно ничего в UI бота (кроме кнопки "Обновить")
- Можно обсуждать баг в чате

### Минусы
- Нет автоматического прикрепления логов/спека
- Нужен отдельный бот (хотя у Faiz есть опыт)
- Telegram → GitHub Issue = доп. прослойка

### Архитектура
```
Telegram бот (Python, на VPS)
  ├─ /bug Сова не кастует МФ
  │   → создаёт GitHub Issue
  │   → Claude фиксит
  │   → бот отвечает "Пофиксил, обнови бота"
  │
  ├─ /status
  │   → показывает открытые баги
  │
  └─ /rollback
      → Claude откатывает последний коммит
```

## Серверная часть: Claude Code авто-фикс

### Как Claude подхватывает баги
**Вариант A — GitHub Actions Webhook:**
```yaml
# .github/workflows/autofix.yml
on:
  issues:
    types: [opened]
jobs:
  fix:
    runs-on: self-hosted  # на VPS Faiz
    steps:
      - uses: actions/checkout@v4
      - run: claude-code fix --issue ${{ github.event.issue.number }}
```

**Вариант B — Cron на VPS:**
```bash
# Каждые 5 минут проверяет новые issues
*/5 * * * * cd /projects/wow-bot && ./check-issues.sh
```

**Вариант C — Telegram бот сам вызывает Claude:**
```python
# При получении /bug команды
async def handle_bug(message):
    issue = create_github_issue(message.text)
    result = subprocess.run(["claude", "fix", f"--issue={issue.number}"])
    if result.returncode == 0:
        build_and_release()
        await message.reply("Пофиксил! Обнови бота.")
    else:
        await message.reply("Не смог пофиксить, передал Faiz.")
```

### Защита от плохих фиксов
1. Claude коммитит в отдельную ветку `fix/issue-{N}`
2. Прогоняет билд — если не собирается, НЕ мержит
3. Создаёт Release только после успешного билда
4. Каждый Release = тег (v1.2.3) → легко откатить
5. "Откатить" = скачать предыдущий Release
