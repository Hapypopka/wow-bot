# Hivemind Rework — архитектура следования и атаки

## Текущие проблемы (корневые)

### 1. FaceUnit дёргается
- Записывает facing в память + TurnStart/TurnStop
- TurnStart начинает поворот, но следующий тик снова пишет facing → дёрганье
- Нужно: записать facing → ждать 1-2 тика → проверить результат

### 2. Следование ненадёжно
- CTM не работает на некоторых клиентах
- FollowUnit работает только <28 ярдов
- MoveForward бежит прямо (не к цели)
- Нет обхода препятствий

### 3. Команды перемешиваются
- "Ко мне" и "Бейте таргет" используют похожий код но разную логику
- Стоп не полностью сбрасывает состояние
- После стопа слейв "залипает" (нельзя ручками управлять)

## Новая архитектура: SlaveStateMachine

Слейв имеет **одно состояние** в каждый момент:

```
enum SlaveState {
    Idle,       // Ничего не делает, ручное управление
    Following,  // Бежит к мастеру
    Attacking,  // Бьёт цель мастера
    Stopped     // Стоп (временный, через 3 сек → Idle)
}
```

### Команды мастера → переходы:
- "Ко мне" → Following
- "Бейте таргет" → Attacking
- "Стоп" → Stopped → (3 сек) → Idle

### Following (следование):
```
Каждый тик (150мс):
  1. Определить позицию мастера (из ObjectManager по GUID)
  2. Рассчитать дистанцию
  3. Если dist > followDistance:
     a. Рассчитать нужный facing (atan2 из мировых координат)
     b. Если |currentFacing - neededFacing| > 0.3 рад:
        - Записать facing в память
        - НЕ двигаться (ждём поворот)
        - Установить флаг _turning = true
     c. Если _turning и уже повёрнуты (< 0.3 рад):
        - _turning = false
        - MoveForwardStart()
     d. Если бежим и повёрнуты — продолжаем бежать
     e. Если бежим но мастер сместился сильно — пересчитать facing
  4. Если dist <= followDistance:
     - MoveForwardStop()
     - Idle ожидание
```

### Attacking (атака):
```
Каждый тик:
  1. AssistUnit(мастер) — взять таргет мастера
  2. Если нет таргета или таргет мёртв → AssistUnit снова
  3. Определить дистанцию до таргета
  4. Если далеко:
     a. FaceUnit к таргету (та же логика поворота)
     b. MoveForward к таргету
  5. Если в castRange:
     - MoveForwardStop()
     - Выполнить ротацию
  6. Хилер: хилить всегда, подбег к мастеру (не к мобу)
```

### FaceUnit (новый, без дёрганья):
```
FaceUnit(player, target):
  1. Рассчитать нужный facing: atan2(target.Y - player.Y, target.X - player.X)
  2. Нормализовать в [0, 2π]
  3. Рассчитать diff = needed - current (кратчайший путь)
  4. Если |diff| < 0.3 рад — уже смотрим, return true
  5. Записать needed в память (мгновенный локальный поворот)
  6. Один раз: TurnLeftStart() или TurnRightStart() (по направлению diff)
  7. Return false (ещё поворачиваемся)

  Следующий тик:
  1. Если |diff| < 0.3 — TurnStop(), return true
  2. Иначе — ничего (WoW сам доворачивает через TurnStart)
```

Ключевое отличие: **не перезаписывать facing каждый тик**. Записать один раз → TurnStart → ждать → TurnStop.

### Стоп:
```
Stop:
  1. MoveForwardStop()
  2. _state = Stopped
  3. Через 3 сек: _state = Idle (полный сброс, ручное управление)
```
